# Spec Cross-Critique

You have independent spec candidates in context from Gemini, Claude, Kimi, and DeepSeek.

Compare them adversarially:

- identify consensus
- identify contradictions
- identify missing acceptance criteria
- identify App Store 4.3 risks
- identify where a candidate would rebuild boilerplate infrastructure unnecessarily
- identify testability gaps
- identify vague or non-verifiable claims

Write `.workflow/iphone-app-factory/spec-cross-critique.md`.

Be strict. A spec is not good enough unless an implementation agent can build and verify it without guessing.
