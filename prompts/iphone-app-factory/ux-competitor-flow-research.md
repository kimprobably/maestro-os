# UX Competitor Flow Research

Research competitor iPhone app flows relevant to this UX Studio run and write the artifact to `.workflow/iphone-app-ux-studio/research/competitor-flows.md`.

## Source Policy

- Use public App Store listings, public websites, public product pages, public screenshots, public videos, and user-provided reference material.
- Do not output secrets, credentials, tokens, cookies, private keys, session values, or environment variable values.
- Do not clone proprietary screens, exact layouts, UI copy, brand identity, screenshots, or assets.
- Treat competitor screens as reference evidence for principles only.

## Required Headings

Use these exact headings:

1. `# Competitor Flow Research`
2. `## Source Policy`
3. `## Source List`
4. `## Competitor Set`
5. `## Flow Patterns`
6. `## Screen Type Coverage`
7. `## UX Strengths`
8. `## UX Weaknesses`
9. `## what_to_adapt`
10. `## what_not_to_copy`
11. `## Reference Entries`

## Required Content

- Include at least four competitor flow references when available.
- For each competitor reference, include app name, source URL or source description, flow name, screen types, evidence summary, `what_to_adapt`, and `what_not_to_copy`.
- Cover setup, primary action, error or blocked state, retention, and settings/accountability flows when relevant.
- Keep source quality explicit: label sources as direct, public listing, public video, review-derived, or inferred.
